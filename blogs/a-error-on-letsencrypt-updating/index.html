a_error_on_letsencrypt_updating

2016-12-07
Server
评论

众所周知，Let’s Encrypt签发的证书的有效期是三个月，而这个博客的证书即将在八月份到期，于是我尝试着手动更新下证书。进入到Let’s Encrypt所在的目录，输入如下代码后出现了一个问题……

./letsencrypt-auto --agree-dev-preview --server https://acme-v01.api.letsencrypt.org/directory certonly
![](/content/images/2016/07/Installing-Python-packages.png) 这一命令卡死在了“Installing Python packages”这里…… 这个问题最常出现在国内的服务器上，稍稍懂行点的人应该能把问题的原因猜得八九不离十了——默认的Python源被墙掉了……嗯，事实的确如此。 既然已经知道了问题所在，那么接下来的任务就很简单，换掉PyPI源即可。很简单，编辑`~/.pip/pip.conf`，如果文件和目录不存在的话，应当先建立文件或者目录，加入或修改以下配置。当然，你也可以把镜像源改成国内其它服务商的。 
[global]
index-url = http://mirrors.aliyun.com/pypi/simple/

[install]
trusted-host=mirrors.aliyun.com
保存即可。 你也可以不修改配置文件，而是在每次安装前临时指定源： 
pip install -i http://< mirror >/simple < package >
如： 
http://mirrors.aliyun.com/pypi/simple ada
