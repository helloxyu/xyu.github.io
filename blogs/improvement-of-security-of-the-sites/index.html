对站点安全性的优化

2016-01-09
Server
评论

注：以下内容所指站点为：the Barrens

启用HSTS
在本站的Nginx配置文件中，加入：

add_header Strict-Transport-Security “max-age=63072000; preload”;

这样做，可以使支持HSTS的浏览器在遇到这个响应头的时候，把当前网站加入HSTS列表，然后在max-age指定的秒数内，当前网站所有请求都会被重定向为https。即使用户主动输入http:或者不输入协议部分，都将重定向到https:地址。这样做可以让浏览器在首次访问本站点后，自动实现http到https的跳转，代替了Web服务器进行301/302重定向的工作。

于是乎，Nginx配置文件中的


return 301 https://$server_name$request_uri;

会在首次访问之后变得多余，因此，可以单独写一个Nginx配置文件，来实现首次访问时http到https的重定向：


server {
        listen 80;
        listen [::]:80 ipv6only=on;
        server_name  zyx.im www.zyx.im;
        return 301 https://$server_name$request_uri;
        server_tokens   off;
}

隐藏服务器版本号信息
观察本博客的请求响应头，会发现这一行server:nginx，而Nginx的具体版本号并没有说明，由于某些 Nginx 漏洞只存在于特定的版本，隐藏版本号可以提高安全性。实现的方法很简单，只需在博客的Nginx配置文件中加入：


server_tokens off;

因为本站使用Ghost，通过Nginx反向代理进行访问，所以还需要加上如下配置：


proxy_hide_header X-Powered-By;

增加https的安全性
为了确保更强的安全性，采取了迪菲－赫尔曼密钥交换：
在/etc/nginx/ssl目录下生成dhparam.pem: 


openssl dhparam -out dhparam.pem 2048

如果你对保密性要求高，而且服务器性能足够强，那么可以生成4096位的加密，本站采用了4096位加密，整个生成过程似乎只用了二十来分钟。

生成完毕后，在配置文件中加入如下内容：


 ssl_prefer_server_ciphers on;
 ssl_dhparam /etc/ssl/certs/dhparam.pem;
 ssl_protocols TLSv1 TLSv1.1 TLSv1.2;
 ssl_ciphers “EECDH+ECDSA+AESGCM EECDH+aRSA+AESGCM EECDH+ECDSA+SHA384 EECDH+ECDSA+SHA256 EECDH+aRSA+SHA384 EECDH+aRSA+SHA256 EECDH+aRSA+RC4 EECDH EDH+aRSA !aNULL !eNULL !LOW !3DES !MD5 !EXP !PSK !SRP !DSS !RC4”;
 keepalive_timeout 70;
 ssl_session_cache shared:SSL:10m;
 ssl_session_timeout 10m;

重启即可。

完成上述工作后，我去SSL Labs测试了一下，结果不错：A+。
